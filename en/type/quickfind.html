QuickFind

QuickFind

When construct a QuickFind, we pass nnn as the capacity of the quick find. We mark the elements in quick find from 000 to n−1n-1n−1 (inclusive).

Different algorithms of quick find bring us different growth of the cost, see below:

Type connected(p, q) union(p, q)
tango.type.QuickFind(N) O(1)\text{O}(1)O(1) O(N)\text{O}(N)O(N)
tango.type.WeightedQuickFind(N) O(lnN)\text{O}(\ln N)O(lnN) O(lnN)\text{O}(\ln N)O(lnN)

connected(p, q): boolean

Returns true iff ppp is connected with qqq, otherwise false.

count(): number

Gets the number of components which are not connected with each other.

union(p, q): number

Unions ppp with qqq and returns the count() after union.
